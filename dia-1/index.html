🎮 Arcade del Código

Misión: Arreglos y el ciclo for. Resuelve los 7 mini retos y el Jefe Final en archivos .js dentro de tu propia carpeta, súbela a GitHub y publícala en GitHub Pages. Cada reto te da el punto de partida y el resultado esperado; tú escribes la solución en tu archivo.

📜 Cómo trabajar cada reto

  1. Lee la misión: dice QUÉ tienes que lograr (no el cómo).
  2. Copia el "punto de partida" (el bloque de código con los datos) a tu archivo .js correspondiente (reto-1.js, reto-2.js…).
  3. Resuélvelo ahí usando console.log. Para ver el resultado, abre tu página y presiona F12 → pestaña Console.
  4. Compara tu salida con el bloque "Deberías ver algo así". ¿Coincide? ¡Subiste de nivel!
  5. ¿Atascado? Recién ahí abre la pista 💡. Primero inténtalo solo: ahí está el aprendizaje.
  6. Marca "¡Misión cumplida!" cuando lo logres. La tarjeta se pondrá verde.

Para probar en tu computador antes de publicar, abre index.html con la extensión Live Server de VS Code (clic derecho → Open with Live Server). Al publicarlo en GitHub Pages funciona directo, sin Live Server.

3 Resuelve cada reto en su archivo .js

En cada reto más abajo tienes su punto de partida (los datos y la tarea en comentarios): cópialo a su archivo y escribe ahí tu solución con console.log. Para ver los resultados, abre tu página y presiona F12 → pestaña Console.

4 Sube tu carpeta a GitHub

Elige uno de los dos caminos:

Opción A · Desde la web (más simple)

  1. Entra a github.com e inicia sesión (o crea tu cuenta).
  2. Arriba a la derecha: + → New repository.
  3. Nombre: practica-arreglos-for. Marca Public y crea el repositorio.
  4. En el repo: Add file → Upload files.
  5. Arrastra TODOS los archivos de tu carpeta (el index.html y los .js).
  6. Escribe un mensaje y haz clic en Commit changes.

Opción B · Con Git (terminal)

Crea primero el repositorio vacío en GitHub y luego, dentro de tu carpeta:

git init
git add .
git commit -m "practica arreglos y for"
git branch -M main
git remote add origin https://github.com/TU-USUARIO/practica-arreglos-for.git
git push -u origin main

5 Publica en GitHub Pages

  1. En tu repositorio, abre Settings (Configuración).
  2. En el menú izquierdo, entra a Pages.
  3. En Source elige Deploy from a branch.
  4. Branch: main · carpeta / (root) → Save.
  5. Espera 1–2 minutos y recarga. Aparecerá tu enlace:
    https://TU-USUARIO.github.io/practica-arreglos-for/
  6. Ábrelo, presiona F12 → Console y verás tus resultados. Ese enlace es tu entrega.
Entrenamiento

Mini retos

Reto 1 · El inventario del explorador Nivel 1

Poder a entrenar: crear un arreglo, acceder por índice y contar.

  1. Muestra el primer objeto de la mochila.
  2. Muestra el último objeto, pero sin escribir el número de la posición.
  3. Muestra cuántos objetos hay en total.
  4. Reemplaza "mapa" por "GPS" y muestra la mochila completa.
¿Necesitas una pista?
  • La primera casilla es la [0], no la [1].
  • El último índice siempre es .length - 1. Así sirve aunque cambie el tamaño.
  • Para reemplazar, asigna directo a esa posición: mochila[posición] = ....
reto-1.js — punto de partida
let mochila = ["cuerda", "linterna", "mapa", "brújula"];

// Tu tarea (resuélvela en este archivo):
// 1) Muestra el PRIMER objeto
// 2) Muestra el ÚLTIMO objeto (usa .length, no el número)
// 3) Muestra cuántos objetos hay
// 4) Cambia "mapa" por "GPS" y muestra la mochila
Primer objeto: cuerda
Último objeto: brújula
Total: 4
Mochila: [ "cuerda", "linterna", "GPS", "brújula" ]

Los textos de las etiquetas los eliges tú: lo que importa son los valores.

Reto 2 · La fila del cajero automático Nivel 1

Poder a entrenar: agregar y quitar elementos (push, unshift, pop, shift, splice).

Haz estos cambios en la fila, en este orden, y muestra la fila después de todo:

  1. Llega "Diego" y se pone al final.
  2. Llega "Eva" con prioridad y se pone al inicio.
  3. Atienden al primero de la fila (se va).
  4. El último se cansa y se va.
  5. La persona que quedó en la posición del medio (índice 1) abandona.
¿Necesitas una pista?
  • "Al final" → push. "Al inicio" → unshift.
  • "Se va el primero" → shift. "Se va el último" → pop.
  • Para sacar uno del medio: splice(inicio, cuántos).
  • Muestra la fila después de cada paso para ir viendo cómo cambia.
reto-2.js — punto de partida
let fila = ["Ana", "Beto", "Carla"];

// Tu tarea (resuélvela en este archivo):
// Sigue los 5 pasos de la misión, en orden,
// y muestra la fila final.
Fila final: [ "Ana", "Carla" ]

Reto 3 · Pasar lista Nivel 2

Poder a entrenar: recorrer un arreglo con el ciclo for.

  1. Recorre la lista de jugadores con un for.
  2. Por cada jugador, muestra "Jugador N: nombre", donde N empieza en 1 (no en 0).
¿Necesitas una pista?
  • Estructura del for: for (let i = 0; i < jugadores.length; i++) { ... }
  • El índice i arranca en 0, pero al humano le muestras i + 1.
  • Usa .length en la condición, nunca un número fijo.
reto-3.js — punto de partida
let jugadores = ["Sora", "Kai", "Luna", "Max"];

// Tu tarea (resuélvela en este archivo):
// Recorre con un for y muestra "Jugador N: nombre"
// (N debe empezar en 1)
Jugador 1: Sora
Jugador 2: Kai
Jugador 3: Luna
Jugador 4: Max

Reto 4 · Poción de doble poder Nivel 2

Poder a entrenar: transformar un arreglo con map.

  1. Crea un arreglo nuevo con cada puntaje multiplicado por 2.
  2. Muestra el arreglo nuevo y también el original, para confirmar que el original NO cambió.
¿Necesitas una pista?
  • map crea un arreglo nuevo aplicando algo a cada elemento.
  • Forma: let nuevo = puntajes.map((p) => ... ).
  • Guarda el resultado en otra variable; el original queda intacto solo.

let puntajes = [10, 25, 40, 55];

// Tu tarea (resuélvela en este archivo):
// 1) Crea un arreglo nuevo con cada puntaje x2
// 2) Muestra el nuevo y el original
Dobles: [ 20, 50, 80, 110 ]
Original: [ 10, 25, 40, 55 ]

Reto 5 · El filtro de aprobados Nivel 2

Poder a entrenar: filtrar con filter y contar con .length.

  1. Arma un arreglo solo con las notas aprobadas (mayor o igual a 4.0).
  2. Muestra ese arreglo y cuántas notas aprobadas hay.
¿Necesitas una pista?
  • filter se queda solo con los elementos cuya condición da true.
  • La condición de aprobado: nota >= 4.0.
  • Para contar, usa .length sobre el arreglo filtrado.
reto-5.js — punto de partida
let notas = [3.5, 6.0, 4.2, 2.8, 7.0, 5.5];

// Tu tarea (resuélvela en este archivo):
// 1) Filtra solo las notas >= 4.0
// 2) Muestra el arreglo filtrado y cuántas son
Aprobadas: [ 6, 4.2, 7, 5.5 ]
Cantidad: 4

Ojo: 6.0 y 7.0 se muestran como 6 y 7 (JavaScript no guarda el cero de adorno).

Reto 6 · El detective de usuarios Nivel 2

Poder a entrenar: buscar con includes, indexOf y find.

  1. ¿Existe el usuario "beto_99"? Muestra true / false.
  2. ¿En qué posición está "carla_07"?
  3. Encuentra y muestra el primer usuario que contenga el texto "_0".
¿Necesitas una pista?
  • "¿Existe?" (true/false) → includes.
  • "¿En qué posición?" → indexOf.
  • "Encuentra el primero que cumple una condición" → find.
  • Para ver si un texto contiene a otro, también sirve .includes("_0") sobre el texto.
reto-6.js — punto de partida
let usuarios = ["ana_22", "beto_99", "carla_07"];

// Tu tarea (resuélvela en este archivo):
// 1) ¿Existe "beto_99"?  (true / false)
// 2) ¿Posición de "carla_07"?
// 3) Primer usuario que contenga "_0"
¿Existe beto_99?: true
Posición de carla_07: 2
Primero con "_0": carla_07

Reto 7 · La caja registradora Nivel 3

Poder a entrenar: acumular todo en un valor con reduce (o con un for + acumulador).

  1. Calcula el total vendido (la suma de todas las ventas).
  2. Calcula el promedio de venta.

Puedes usar reduce o un for con un acumulador. ¡Las dos formas valen!

¿Necesitas una pista?
  • Con reduce: ventas.reduce((acum, v) => acum + v, 0). El 0 es el punto de partida.
  • Con for: crea let total = 0 ANTES del ciclo y adentro haz total += ventas[i].
  • El promedio es el total dividido por ventas.length.
reto-7.js — punto de partida
let ventas = [3000, 5500, 1200, 8000];

// Tu tarea (resuélvela en este archivo):
// 1) Calcula el total vendido
// 2) Calcula el promedio
Total: 17700
Promedio: 4425
Jefe Final

El gran reto integrador

🏆 Reto Final · El Torneo de la Academia Jefe Final

Acá se junta TODO: índices, for, acumulador, máximo, filter, sort y arreglos paralelos.

Tienes dos arreglos paralelos: en la misma posición, el nombre y su puntaje. Resuelve en orden:

  1. Muestra cuántos jugadores compiten.
  2. Con un for, muestra "nombre: puntaje" de cada jugador.
  3. Calcula el puntaje total y el promedio.
  4. Encuentra el puntaje máximo.
  5. Arma una lista con los puntajes que superan el promedio.
  6. Muestra el ranking de puntajes ordenados de mayor a menor.
  7. Bonus: muestra el nombre del ganador (el del puntaje máximo).
¿Necesitas una pista?
  • Paso 2: en el for, usa el mismo i para los dos arreglos: nombres[i] y puntajes[i].
  • Paso 3: total con reduce o for; promedio = total / cantidad.
  • Paso 4: el máximo lo sacas con un for comparando, o con Math.max(...puntajes) (los ... "abren" el arreglo).
  • Paso 5: filter con la condición p > promedio.
  • Paso 6: sort((a, b) => b - a) ordena de mayor a menor. ⚠️ sort() a secas ordena mal los números.
  • Paso 7: encuentra la posición del máximo con indexOf(maximo) y usa ese índice en nombres.
torneo-final.js — punto de partida
let nombres  = ["Sora", "Kai", "Luna", "Max", "Nia"];
let puntajes = [120, 85, 200, 60, 150];

// Tu tarea (resuélvela en este archivo):
// Resuelve los 7 pasos de la misión, en orden.
// Tip: hazlo paso por paso y abre tu página (F12) para ir revisando.
Total de jugadores: 5
Sora: 120
Kai: 85
Luna: 200
Max: 60
Nia: 150
Puntaje total: 615
Promedio: 123
Puntaje máximo: 200
Sobre el promedio: [ 200, 150 ]
Ranking: [ 200, 150, 120, 85, 60 ]
🏆 Ganador: Luna
